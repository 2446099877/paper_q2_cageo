# C&G 一次性补全信息表

正式投稿前，只需要把下面信息补齐一次，我就可以继续把 `C&G` 主稿、cover letter、code availability 和提交包一起收口。

## 直接复制后填写

```text
1. 作者信息
- 作者1英文名（名 / 姓）：
- 作者2英文名（名 / 姓）：
- 作者顺序是否就是上面顺序：
- 通讯作者是谁：
- 通讯作者邮箱：
- 每位作者 ORCID（没有就写无）：

2. 单位信息
- 学院/系/实验室英文名：
- 学校/单位英文全称：
- 城市英文名：
- 省份英文名（没有可留空）：
- 国家英文名：

3. 经费与声明
- 是否有基金：
- 如果有，基金英文全称 + 编号：
- 是否确认无利益冲突（是/否）：

4. 代码与仓库
- 是否接受论文接收后公开代码（是/否）：
- 仓库链接（如果现在还没有，就写“后补”）：
- 审稿阶段是否愿意提供 reviewer-safe 仓库/压缩包（是/否）：

5. 作者贡献
- 作者1主要贡献：
- 作者2主要贡献：
- 如果还有其他作者，请逐个补充：
```

## 默认规则

- 如果你写 `无基金`，我会保留当前 `no external funding` 口径
- 如果你写 `仓库后补`，我不会提前写死错误链接
- 如果作者贡献你不想自己组织措辞，我会按当前仓库里的真实工作内容生成一版保守英文版本
